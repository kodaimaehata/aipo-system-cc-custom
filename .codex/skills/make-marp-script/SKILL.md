# make-marp-script

## Description
構成案（Outline）と材料束（Input Bundle）から、プレゼンテーション用の台本（スピーカーノート付き）を生成します。

これはMarpスライド作成の**Phase 2**です。

## Usage
**Triggers**: `/marp-script`, "台本を作成", "スクリプトを生成"

## Inputs

| パラメータ | 説明 | 必須 | ソース |
|-----------|------|------|--------|
| outline | Outlineの内容またはファイルパス | Yes | Phase 1出力 |
| input_bundle | Input Bundleの内容またはファイルパス | Yes | Phase 1出力 |
| style | 話し方スタイル（conversational/formal/storytelling） | No | conversational |
| output_dir | 出力ディレクトリ | No | ./marp-output |

## Instructions

### Step 1: 入力の取得
1. Outlineを読み込む（ファイルパスまたは直接入力）
2. Input Bundleを読み込む（ファイルパスまたは直接入力）
3. 話し方スタイルを確認

### Step 2: 構造の確認
Outlineの各セクションを確認:
- 時間配分
- 各セクションの狙い
- 含めるべきポイント

### Step 3: HITL - オープニングスタイル選択

**ユーザーに選択を求める**:

```
オープニングのスタイルを選択してください:

[A] 質問から始める
    「〇〇で困ったこと、ありませんか？」のように問いかけで始める

[B] 驚きの事実から始める
    インパクトのある数字や事実を提示して興味を引く

[C] ストーリーから始める
    「先日こんなことがありました...」と体験談から始める

選択 (A/B/C): _
```

### Step 4: HITL - クロージングスタイル選択

**ユーザーに選択を求める**:

```
クロージングのスタイルを選択してください:

[A] 質疑応答への誘導
    「ご質問があればぜひ」と対話を促す

[B] 強いCTAで締める
    「今日からぜひ〇〇してください」と行動を促す

[C] 余韻を残す
    印象的なメッセージや引用で締める

選択 (A/B/C): _
```

### Step 5: 台本生成
各スライドに対して以下の形式で台本を生成:

```markdown
# {theme} - 台本

## スライド1: {タイトル}

**表示内容**:
- ポイント1
- ポイント2

**話す内容（約X分）**:
「（ここに実際に話す言葉をそのまま書く）

〜〜〜〜〜〜〜〜〜〜

（間を取る）

〜〜〜〜〜〜〜〜〜〜」

**演出メモ**:
- 間を取るタイミング
- ジェスチャーや動き
- スライド切り替えのキュー

---

## スライド2: {タイトル}
...
```

### Step 6: 台本の品質チェック
以下を確認:
- [ ] 各スライドの時間配分が適切か
- [ ] 話し言葉として自然か
- [ ] トーンが一貫しているか
- [ ] 聴衆に合った言葉遣いか

### Step 7: ファイル出力
1. `{output_dir}/script.md` に台本を保存
2. ユーザーに完了を報告

## Output Format

**成果物**:
- `script.md` - プレゼンテーション台本

**台本の構成**:
```markdown
# {theme} - 台本

## メタ情報
- 総時間: X分
- スライド枚数: X枚
- オープニング: {選択したスタイル}
- クロージング: {選択したスタイル}

---

## スライド1: タイトル
**表示**: ...
**話す**: 「...」
**演出**: ...

---

## スライド2: ...
...
```

**次のステップ**:
- `/marp-slide` を実行してMarp Markdownを生成

## HITL Integration Points

1. **オープニングスタイル選択**: A/B/C から選択
2. **クロージングスタイル選択**: A/B/C から選択
3. **台本レビュー**: 生成後にユーザー確認

## Tips

- 話し言葉は「です・ます」調を基本に
- 間の取り方を明記（カッコで注記）
- スライド切り替えのタイミングを明示
- 想定質問への回答メモも追記可能
