# Pattern Dictionary

| Pattern | Use When | Avoid When | Core Blocks | Fallback |
|---|---|---|---|---|
| cover | opening slide, expectation-setting | too much body content | title, subtitle, brand/photo | dense brief cover |
| agenda | section list and reading order | each section needs long explanation | title, section list, optional short note | grouped agenda |
| comparison | 2-3 options, trade-offs, before/after | too many comparison axes | title, lead, left/right columns | comparison table |
| flow | ordered steps, process, cause/effect | long chronology or heavy prose | title, lead, step sequence | vertical flow |
| graph | trend, ranking, composition | exact lookup is primary | title, lead, graph type recommendation, takeaway | compact table |
| table | dense lookup, many metrics, exact values | opening slides or emotional message | title, lead, table, highlighted cells | comparison table |
| parallel | 2-4 peer elements with equal weight | ordering matters or too many elements | title, lead, equal cards/columns | compare cards |

## Pattern Notes

- Keep every pattern inside the `1 slide = 1 message` rule.
- For graph and table, recommendation stops at representation choice.
- For comparison and parallel, preserve visual symmetry unless the message is asymmetry.
- `Fallback` must always be a concrete alternative layout or layout variant, not an editing instruction.
